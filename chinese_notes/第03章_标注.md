# 第 3 章 标注 (Labeling)

## 3.1 写作动机

第 2 章产出的特征矩阵 $X$,还需要给每行配上标签 $y_i$,才能用于监督学习。本章介绍财金数据特有的标注方法。

## 3.2 固定时间窗口法 (The Fixed-Time Horizon Method)

最常见但**有缺陷**的方法:观察 $h$ 步后的收益,根据预定阈值 $\tau$ 划分:

$$y_i = \begin{cases} -1 & r_{t_{i,0}, t_{i,0}+h} < -\tau \\ 0 & |r_{t_{i,0}, t_{i,0}+h}| \leq \tau \\ +1 & r_{t_{i,0}, t_{i,0}+h} > \tau \end{cases}$$

**两大缺陷:**
1. 时间条本身统计性质差。
2. 阈值 $\tau$ 是常量,但波动率不是——夜盘波动率 $\sigma=10^{-4}$,开盘波动率 $\sigma=10^{-2}$,同一阈值会导致绝大多数样本标为 0,即使收益其实显著。

## 3.3 计算动态阈值

正确做法:阈值随波动率变化。Snippet 3.1 用日级 EWM 标准差估计:

```
def getDailyVol(close, span0=100):
    df0 = close.index.searchsorted(close.index - pd.Timedelta(days=1))
    df0 = close.loc[close.index[df0[df0>0]]] / \
          close.loc[close.index[df0[df0>0]-1]].values - 1
    return df0.ewm(span=span0).std()
```

## 3.4 三道闸法 (The Triple-Barrier Method) ★

作者首创的标注方法。每个观察点设置三个屏障:

- **上水平屏障**: 止盈线,触发标 $+1$
- **下水平屏障**: 止损线,触发标 $-1$
- **垂直屏障**: 持仓时间上限,触发标 $\mathrm{sign}(\text{return})$ 或 $0$

**特点:**
- **路径相关**: 与策略实际操作一致(止盈、止损、超时)。
- **波动率自适应**: 水平屏障宽度 = `pt*trgt` 和 `sl*trgt`,trgt 是动态波动率。
- **8 种屏障组合**:`[1,1,1]`(标准)、`[0,1,1]`(无止盈)、`[1,1,0]`(无超时)等等。

### 实现要点

| 函数 | 功能 |
|------|------|
| `applyPtSlOnT1` | 给定每个事件,计算最早触碰的屏障时间 |
| `getEvents` | 从触发事件 (`tEvents`) 出发,生成 events 表 |
| `addVerticalBarrier` | 添加 N 天后的垂直屏障 |
| `getBins` | 把 events 转化为最终标签 `bin` |

## 3.5 学习方向与大小

如果模型既要预测**方向**(side: long/short)又要预测**大小**(size: bet 大小),通常需要对称水平屏障。`getEvents` 在 `side=None` 时执行此模式。

## 3.6 元标注 (Meta-Labeling) ★★★

**问题:** 假设你已经有一个一级模型决定方向(long/short),只需要决定要不要下注、下多大注。

**元标注做法:**
1. 一级模型(可以是基本面模型、技术指标、甚至 PM 直觉)决定方向 `side`。
2. 用三道闸法生成"如果按这个方向下注,赚还是赔"的标签 `bin ∈ {0, 1}`。
3. 训练**二级模型**(ML)预测 P(bin=1),用此概率决定头寸规模。

**优势:**
1. 把基本面模型变成 ML 模型,可解释性 + ML 精度。
2. 限制过拟合范围——ML 不决定方向,只决定大小。
3. 提升 F1-Score:一级模型给高召回率,元标注模型给高准确率。
4. 单独建模 long/short——做多和做空的特征可以不同。

**核心见解:** "决定开仓"和"决定大小"是两个独立问题,分开建模更稳健。

## 3.7 如何使用元标注

ROC 曲线中,提高真阳率会带来更多假阳。元标注用一个二级 ML 过滤一级模型的假阳——尤其当一级模型召回率高但精度低时,元标注能显著提升 F1-Score。

## 3.8 Quantamental 之路

许多顶尖对冲基金正在做"基本面 + 量化"的结合(quantamental)。元标注是天然桥梁:让基本面/直觉模型决定方向,让 ML 决定行动与否。

可以监控 PM 的睡眠、压力等生理指标,用元标注预测他们什么时候判断更可靠。

## 3.9 删除不必要的标签

某些 ML 分类器在类别极度不均衡时表现很差。Snippet 3.8 (`dropLabels`) 递归删除占比 < `minPct` 的稀有标签:

```python
def dropLabels(events, minPct=.05):
    while True:
        df0 = events['bin'].value_counts(normalize=True)
        if df0.min() > minPct or df0.shape[0] < 3:
            break
        events = events[events['bin'] != df0.idxmin()]
    return events
```

## 3.11 本章核心要点

1. **不要**使用固定时间窗口 + 固定阈值的标注方式。
2. 使用**三道闸法**:与实际交易行为一致,波动率自适应。
3. **元标注**让你把现有的一级模型(基本面/技术分析/直觉)变成 ML,只让 ML 决定大小。
4. 类别极度不均衡时,删除稀有标签。
