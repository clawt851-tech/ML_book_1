# 第 9 章 用交叉验证做超参数调优

## 9.1 写作动机

超参调优做不好,模型必然过拟合,实盘表现一定让人失望。第 7 章说过金融 CV 极难——本章用 Purged K-Fold CV 来调超参。

## 9.2 网格搜索 CV

`GridSearchCV` 对所有参数组合做穷举搜索。要把第 7 章的 `PurgedKFold` 作为 `cv` 参数传入,防止信息泄漏。

`Snippet 9.1: clfHyperFit` 实现了:
1. 用 `PurgedKFold` 做内层 CV 找最佳参数。
2. 可选用 `BaggingClassifier` 包装最佳估计器,在全数据上重训(降低方差)。

### sklearn Pipeline 的 bug

sklearn 的 `Pipeline.fit()` **不接受** `sample_weight` 参数。`Snippet 9.2: MyPipeline` 子类把 `sample_weight` 转成 `fit_params`,绕过这个 bug。

## 9.3 随机搜索 CV

参数太多时,网格搜索不可行。用 `RandomizedSearchCV` 从分布里采样。两个优势:
1. 控制总搜索量(预算可控)。
2. 不重要参数不会浪费搜索次数。

### 9.3.1 Log-Uniform 分布

SVM 的 `C`、`gamma` 等参数对**对数尺度**变化敏感(从 0.01 到 1 跟从 1 到 100 的影响相当)。所以应该让 $\log[x] \sim U[\log a, \log b]$:

CDF: $F[x] = \frac{\log[x] - \log[a]}{\log[b] - \log[a]}$, $a \leq x \leq b$

PDF: $f[x] = \frac{1}{x \log[b/a]}$, $a \leq x \leq b$

`Snippet 9.4: logUniform_gen` 在 scipy 里实现这个分布。

## 9.4 评分函数选择

`Snippets 9.1/9.3` 中:
- **元标注 (binary 0/1)**: 用 `scoring='f1'`(高假阳类不平衡时,accuracy 会被高 P[0] 主导)。
- **其他**: 用 `scoring='neg_log_loss'`(交叉熵)。

### 为什么 neg_log_loss 优于 accuracy

投资策略**靠高置信度的正确预测赚钱**:
- accuracy 对所有错误等同惩罚——但低置信度错误亏小,高置信度错误亏大。
- log loss 用预测概率算损失:

$$L[Y, P] = -N^{-1} \sum_n \sum_k y_{n,k} \log p_{n,k}$$

- 加权后的 log loss = 头寸大小加权 PnL 的概率版本。

**总结:** 调超参用 `neg_log_loss`,不要用 `accuracy`。

## 9.6 本章核心要点

1. 调超参的 CV **必须用 Purged K-Fold**,绝不能用普通 KFold。
2. sklearn `Pipeline` 不接受 `sample_weight`——用 `MyPipeline` 子类绕过。
3. 大搜索空间用随机搜索,而不是网格搜索。
4. 对 SVM 的 C、gamma 等"乘性参数",用 log-uniform 而不是 uniform。
5. 投资 ML 用 `neg_log_loss` 评分,不要用 `accuracy`。
