# 第 6 章 集成方法

## 6.1 写作动机

集成学习方法(bagging / boosting / RF)在一般 ML 教材里都有,本章不再赘述基础。重点是**讲清楚它们为什么有效,以及在金融场景中怎么用才不会用错**。

## 6.2 三种误差来源

任何回归/分类问题的均方误差可分解为:

$$E\left[(y_i - \hat{f}[x_i])^2\right] = \underbrace{(E[\hat{f}[x_i]] - f[x_i])^2}_{\text{bias}^2} + \underbrace{V[\hat{f}[x_i]]}_{\text{variance}} + \underbrace{\sigma_\varepsilon^2}_{\text{noise}}$$

| 来源 | 含义 | 什么时候大 |
|------|------|-----------|
| Bias | 模型假设过强 | 欠拟合 |
| Variance | 对训练集敏感 | 过拟合 |
| Noise | 不可降低的随机性 | 数据本身噪声 |

**集成方法的目标:** 通过组合多个弱学习器减少偏差和/或方差。

## 6.3 Bootstrap 聚合 (Bagging)

### 步骤
1. 有放回抽样产生 N 个训练子集。
2. 每个子集独立训练一个估计器。
3. 最终预测 = N 个预测的均值(回归)或多数投票(分类)。

### 6.3.1 方差缩减

bagged 预测方差:

$$V\left[\frac{1}{N}\sum_{i=1}^N \varphi_i\right] = \bar{\sigma}^2 \left(\bar{\rho} + \frac{1 - \bar{\rho}}{N}\right)$$

- 当 $\bar{\rho} \to 1$,V 趋向 $\bar{\sigma}^2$,bagging **失效**(高度相关 = 没有多样性)。
- 序列 bootstrap(第 4 章)的目标就是降低 $\bar{\rho}$。

### 6.3.2 准确率提升

二分类多数投票下,只要单分类器 $p > 1/k$ 且 N 足够大,bagging 准确率会**严格大于** $p$(Condorcet 陪审定理)。**反过来**:如果单分类器太差($p \ll 1/k$),bagging 也无法挽救——bagging 只擅长降方差,不擅长降偏差。

### 6.3.3 观察冗余

- 重叠 → bootstrap 子集高度相似 → $\bar{\rho} \approx 1$ → bagging 无效。
- OOB 准确率被严重高估,因为 OOB 样本看起来和 in-bag 几乎一样。
- 解决方案:`max_samples = avgU`(第 4 章),或序列 bootstrap。

## 6.4 随机森林 (RF)

RF = bagging + **特征随机抽样**。每个节点分裂只考虑随机子集的特征,这进一步降低了 $\bar{\rho}$。

**金融场景三个常见错误及对应修复:**
1. **样本冗余** → `max_samples = avgU`,即 sklearn 中 `BaggingClassifier(max_samples=avgU)` 包装一棵单树 RF。
2. **类别不均衡** → `class_weight='balanced_subsample'`。
3. **早停** → `min_weight_fraction_leaf=0.05` 等。

### Snippet 6.2 给出三种等价的 RF 设置方式

```python
clf0 = RandomForestClassifier(n_estimators=1000,
                               class_weight='balanced_subsample',
                               criterion='entropy')
clf1 = DecisionTreeClassifier(criterion='entropy', max_features='auto',
                               class_weight='balanced')
clf1 = BaggingClassifier(base_estimator=clf1, n_estimators=1000,
                         max_samples=avgU)
clf2 = RandomForestClassifier(n_estimators=1, criterion='entropy',
                              bootstrap=False,
                              class_weight='balanced_subsample')
clf2 = BaggingClassifier(base_estimator=clf2, n_estimators=1000,
                         max_samples=avgU, max_features=1.)
```

## 6.5 提升 (Boosting)

| 特性 | Bagging | Boosting |
|------|---------|----------|
| 训练方式 | 并行 | 顺序 |
| 弱学习器丢弃 | 否 | 是(精度低于阈值) |
| 样本权重 | 相同 | 误分样本权重提升 |
| 解决 | 方差 | 偏差 + 方差 |
| 过拟合风险 | 低 | 高 |

## 6.6 Bagging vs. Boosting in Finance

**金融数据低信噪比 → 过拟合是主要敌人 → bagging 优于 boosting。**

且 bagging 可并行,boosting 必须顺序——计算效率上 bagging 也有优势。

## 6.7 Bagging 用于扩展性

SVM 等不适合大数据集的算法可以通过 bagging 拆解为多个小问题。配合 `tol=1e-3` 等早停参数,可在大数据上获得快速稳健估计。

## 6.9 本章核心要点

1. **金融选 bagging 不选 boosting**:低信噪比 + 过拟合优先级 + 并行优势。
2. RF 中务必设置 `max_samples=avgU` 防止冗余样本作怪。
3. 类别不均衡用 `class_weight='balanced_subsample'`。
4. 不要相信 OOB 准确率——做 stratified k-fold(无 shuffle)的 CV。
5. SVM 大数据用 bagging 包装 + 早停。
