# 第 8 章 特征重要性

## 8.1 写作动机

> **Marcos 第一回测定律:** "Backtesting is not a research tool. Feature importance is."
>
> *回测不是研究工具,特征重要性才是。*

金融研究最普遍的错误是:跑数据 → ML → 回测,反复修改直到回测好看为止。这就是**通过回测做研究**——必然导致虚假发现。

**正解:** 在做任何回测之前,先研究特征重要性,理解 ML 模型为什么有效。

## 8.2 特征重要性的重要性

- 解开 ML 黑箱:理解算法找到了什么模式。
- 删除噪声特征,简化模型。
- 跨资产验证:重要特征是否在其他金融工具上同样重要?
- 制度切换:重要性会随市场状态变化吗?

## 8.3 有替代效应的方法 (Substitution Effects)

替代效应 = 多重共线性的 ML 版本——两个相关特征会"瓜分"重要性。

### 8.3.1 Mean Decrease Impurity (MDI)

- **定义:** 树形分类器中,每个节点用某特征做分裂带来的不纯度下降,在森林上累加平均。
- **特点:** 仅适用于树模型;in-sample;`feature_importances_` 之和 = 1。
- **注意:**
  - 设 `max_features=int(1)` 防止 mask 效应——每层只看一个特征。
  - 把 0 替换成 `np.nan`(0 表示未被选中,而不是无价值)。
  - MDI 受替代效应影响——两个相同特征会各分得一半。

### 8.3.2 Mean Decrease Accuracy (MDA)

- **定义:** 训练完模型后,逐个 shuffle 每列特征,看 OOS 性能下降多少。
- **特点:** 任何分类器适用;OOS;**必须**在 purged k-fold 中做。
- **替代效应:** 仍然存在——shuffle 一个特征,另一个相同特征仍能预测。

## 8.4 无替代效应的方法

### 8.4.1 Single Feature Importance (SFI)

- **定义:** 每次只用一个特征训练,记录 OOS 性能。
- **特点:** 无替代效应;但**忽视联合效应**——某些特征只在配合其他时有用。

## 8.5 平行 vs. 堆叠特征重要性

两种研究路径:

| 方法 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **平行** (Parallelized) | 对每个工具单独跑特征重要性,聚合 | 并行快 | 排名因替代效应跨工具波动 |
| **堆叠** (Stacked) | 把所有工具数据标准化后合并,跑一次 | 数据多、抗噪、抗过拟合 | 内存计算开销大 |

作者**强烈推荐堆叠**——结论更具普遍性。

## 8.7 PCA 与特征重要性的对照

- PCA 是无监督的,不知道标签 → 不会过拟合分类标签。
- 用 weighted Kendall's tau 比较"特征重要性排名"与"PCA 排名(逆序)"。
- 高相关 → 说明 ML 找到的模式与数据本征结构一致 → 不太可能是过拟合。

## 8.9 本章核心要点

1. **回测不是研究工具,特征重要性才是。**
2. 三种方法各有所长:MDI 快、MDA 通用、SFI 无替代效应。
3. PCA 与监督特征重要性的一致性是"非过拟合"的有力旁证。
4. 优先**堆叠特征重要性**而不是单独工具,排名更稳。
5. 在做任何回测之前,先看哪些特征重要、为什么重要。
