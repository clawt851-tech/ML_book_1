# 第 4 章 样本权重

## 4.1 写作动机

ML 教科书几乎都假设观察是 IID(独立同分布),但金融时间序列**根本不是 IID**——这是大量 ML 应用在金融领域失败的重要原因之一。本章介绍如何用样本加权与采样修正纠正非 IID 带来的偏差。

## 4.2 重叠的标注 (Overlapping Outcomes)

第 3 章给每个特征 $X_i$ 配上了基于价格区间 $[t_{i,0}, t_{i,1}]$ 的标签 $y_i$。当 $i < j$ 但 $t_{j,0} < t_{i,1}$ 时,$y_i$ 和 $y_j$ 会同时依赖于公共收益 $r_{t_{j,0}, \min\{t_{i,1}, t_{j,1}\}}$——这就是非 IID 的来源。

**比喻:** 你想从 100 个病人的血液样本中预测胆固醇。但实验室助理把每个试管的血泼到了右侧 9 个试管里——现在试管 10 含有病人 1-10 的血。这就是金融数据的真实情况,而且"泼洒模式"非确定且未知。

## 4.3 并发标签数 (Number of Concurrent Labels)

**定义:** 两个标签 $y_i, y_j$ 在 $t$ 时刻**并发**,当且仅当它们都依赖于至少一个共同收益 $r_{t-1,t}$。

定义指示变量 $1_{t,i} \in \{0, 1\}$($X_i$ 的标签生命周期是否覆盖 $t$),则 $t$ 时刻并发标签数:

$$c_t = \sum_{i=1}^{I} 1_{t,i}$$

`Snippet 4.1: mpNumCoEvents` 实现了这个统计。

## 4.4 标签的平均唯一性 (Average Uniqueness)

某个标签 $i$ 在 $t$ 时刻的唯一性:$u_{t,i} = 1_{t,i} c_t^{-1}$。

整条标签的平均唯一性:

$$\bar{u}_i = \frac{\sum_{t=1}^{T} u_{t,i}}{\sum_{t=1}^{T} 1_{t,i}}$$

可解释为该标签生命周期上 $c_t$ 的调和平均的倒数。

## 4.5 自助法分类器与唯一性

### 4.5.1 序列自助法 (Sequential Bootstrap)

标准 bootstrap 假设 IID,在金融数据上会导致 in-bag 样本之间互相冗余,out-of-bag 样本与 in-bag 极其相似——这会**严重高估** out-of-bag 准确率。

**序列自助法**:每次抽样后,**降低**与已选样本高度重叠的样本被再次选中的概率。

具体步骤:
1. 第一次抽样均匀:$\delta_i^{(1)} = I^{-1}$。
2. 假设第一次选中了 $i$。把 $\{i\}$ 加入 $\varphi$,重新计算每个 $j$ 在 $\varphi$ 下的平均唯一性 $\bar{u}_j^{(2)}$。
3. 第二次抽样概率: $\delta_j^{(2)} = \bar{u}_j^{(2)} / \sum_k \bar{u}_k^{(2)}$。
4. 重复直到选中 $I$ 个样本。

蒙特卡洛实验显示:序列自助法的样本平均唯一性中位数 ≈ 0.7,而标准 bootstrap ≈ 0.6——**统计上显著更接近 IID**。

## 4.6 收益归因 (Return Attribution)

序列 bootstrap 解决"采样接近 IID"的问题,但训练时还需要**给每个样本赋权**——重叠多的样本权重该低,绝对收益大的样本权重该高。

**作者推荐的样本权重:**

$$\tilde{w}_i = \left| \sum_{t=t_{i,0}}^{t_{i,1}} \frac{r_{t-1,t}}{c_t} \right|, \quad w_i = \tilde{w}_i I \left(\sum_{j=1}^{I} \tilde{w}_j\right)^{-1}$$

意思是:在每个时间点把当时的收益按并发数 $c_t$ 平摊给所有还活着的标签;再把这一标签生命周期上的份额累加。最后归一化使权重和等于 $I$(因为 sklearn 默认权重为 1)。

## 4.7 时间衰减 (Time Decay)

市场是适应性系统(Lo 2017),旧观察价值低于新观察。Snippet 4.11 实现分段线性时间衰减:

- $c = 1$:无衰减
- $0 < c < 1$:线性衰减,但所有样本仍有正权重
- $c = 0$:权重线性收敛到 0
- $c < 0$:最旧的 $|c|$ 比例样本权重为 0(被遗忘)

衰减按"累计唯一性"而非时间——避免对冗余观察衰减过快。

## 4.8 类别权重 (Class Weights)

不同于样本权重,类别权重用于纠正不均衡分类。在金融中,稀有事件(如闪崩)往往才是最有价值的。sklearn 中:

```python
RandomForestClassifier(class_weight='balanced')
RandomForestClassifier(class_weight='balanced_subsample')  # 推荐用于 bagging
```

## 4.10 本章核心要点

1. 金融标签**不是 IID**——存在重叠。
2. 标准 bootstrap 在重叠数据上失败:用**序列自助法**让样本接近 IID。
3. 训练时再用**收益归因**计算样本权重(重叠少 + 绝对收益大 = 高权重)。
4. 添加**时间衰减**让旧样本权重逐步降低。
5. 不均衡分类问题,使用 `class_weight='balanced'`。
