# 第 16 章 机器学习资产配置

## 16.1 写作动机

Markowitz 1952 年的均值-方差优化是金融经济学奠基,但实践中有"诅咒"——不稳定、对输入误差极度敏感。本章用机器学习方法做更稳健的资产配置:**Hierarchical Risk Parity (HRP)**。

## 16.2 凸优化的问题

均值-方差(以及最小方差)需要协方差矩阵求逆。但金融协方差矩阵往往**条件数差**——少量噪声导致权重剧烈摆动。

## 16.3 Markowitz 诅咒

资产相关性越高 → 协方差矩阵越接近奇异 → 求逆越不稳定 → MV 配置越极端。讽刺的是,**最该用分散的时候(高度相关),MV 反而最不可靠**。

## 16.4 从几何到层级关系

MV 把每个资产视作"图中的孤立节点"——所有节点彼此相连,无层级。但金融资产实际上有自然的层级聚类(股 → 板块 → 子板块 → 公司)。

### 16.4.1 树状聚类

第 1 步: 把相关矩阵转化为距离矩阵:

$$d_{i,j} = \sqrt{\frac{1 - \rho_{i,j}}{2}}$$

第 2 步: 对距离做 single-linkage 层级聚类。

### 16.4.2 拟对角化 (Quasi-Diagonalization)

对协方差矩阵的行列重排,使相似资产相邻——大值聚集在对角线附近。

### 16.4.3 递归二分 (Recursive Bisection)

```
权重 w = 1 (所有资产)
while 仍有可分集合:
    把当前集合从中点二分为 c0 和 c1
    在 c0、c1 内分别用 inverse-variance 求子集方差 V0、V1
    分配权重: alpha = 1 - V0 / (V0 + V1)
    w[c0] *= alpha
    w[c1] *= 1 - alpha
    递归处理 c0 和 c1
```

**优点:**
- **不需协方差求逆**——避免 Markowitz 诅咒。
- 输出权重平稳、可解释、与层级一致。
- 蒙特卡洛实验显示 HRP 的 OOS 方差**优于 MV 和 IVP**。

## 16.5 数值示例

10 个资产分两组(每组内高相关、组间低相关):
- HRP 给两组各分约一半,组内按 inverse variance 分配。
- IVP 完全忽略相关结构。
- MV 在多个资产上"all-in",高度集中。

## 16.6 OOS 蒙特卡洛模拟

| 方法 | OOS 方差(相对 HRP) |
|------|---------------------|
| HRP | 1.00 |
| IVP | ~1.10 |
| MinVar (Markowitz) | ~1.50 |

HRP 持续优于 MinVar,尤其在高维场景。

## 16.7 进一步研究

可以尝试别的距离度量(基于 information theory, copula 等),或别的 linkage(complete, average)。

## 16.8 结论

HRP 是金融 ML 的标志性贡献——直接挑战教科书传统。

## 16.10 本章核心要点

1. **Markowitz 在实践中常常有害**——尤其当资产相关性高时。
2. **HRP** 不需求逆,基于层级聚类 → 稳定、可解释、OOS 表现好。
3. 三步法:相关距离 → 层级聚类 → 拟对角化 → 递归二分。
4. 用 IVP 作中间基准:HRP 介于 IVP(忽略相关)与 MinVar(完全考虑相关)之间,实践效果最好。
